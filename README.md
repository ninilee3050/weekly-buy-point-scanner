# MMRM 주봉 매수후보 탐지 도구

MMRM 전략(MACD·Momentum·RSI·MFI)의 주봉 매수후보 조건을 과거 확정 주봉과 이번 주 진행 중인 주봉에서 탐지하는 Python Tkinter 스캐너입니다.

이 프로그램은 자동매매 프로그램이 아니라, 문서에 적힌 주봉 매수 시나리오를 코드가 정확히 이해했는지 확인하기 위한 도구입니다.

사용자는 Investing.com 주봉 차트를 기준 화면으로 보고 있으며, 프로그램은 그 화면과 최대한 같은 주봉 기준으로 MMRM 매수후보를 검출하는 것을 목표로 합니다.

이 도구는 매매 추천, 자동매매, 실시간 주문 프로그램이 아닙니다. 전략 문서의 조건을 코드가 제대로 해석하는지 확인하기 위한 탐지용 프로그램입니다.

## 현재 목적

이 프로그램의 v1 목표는 단 하나입니다.

`MMRM 매수후보 조건을 주봉 데이터에서 정확히 찾아낸다.`

주요 사용 방식은 두 가지입니다.

* 개별 티커 검색: 과거 확정 주봉에서 MMRM 매수후보가 언제 나왔는지 확인
* Top100 이번주 스캐너: 미국 시가총액 상위 종목 중 이번 주에 새 MMRM 매수후보가 생겼는지 확인

핵심 목적은 다음과 같습니다.

* 티커를 입력하면 주봉 데이터를 불러옵니다.
* Momentum, MACD, RSI, MFI를 계산합니다.
* 문서에 정의한 MMRM 주봉 매수 시나리오에 맞는 매수후보 날짜를 찾습니다.
* 화면에는 매수후보 표만 보여줍니다.
* 결과 CSV를 `outputs/` 폴더에 저장합니다.

이 README에서는 전략 조건에 맞는 지점을 `매수후보`라고 부릅니다.

다만 기존 코드와 CSV 파일명에는 구현 이력상 `buy_points`, `매수포인트`, `매수포인트날짜` 표현이 일부 남아 있을 수 있습니다. 이 표현들은 확정 수익 지점이 아니라, MMRM 시나리오 조건에 맞는 후보 지점을 의미합니다.

## 실행 방법

프로그램 실행은 `실행하기.bat`을 더블클릭하면 됩니다.

이 파일은 CMD창을 남기지 않고 GUI 프로그램만 실행하도록 `pythonw`로 `app.py`를 실행합니다.

직접 GUI를 실행하려면 아래처럼 실행할 수도 있습니다.

```bash
python app.py
```

### 화면 없이 자동 스캔 실행

GitHub Actions 같은 자동실행 환경에서는 GUI 창을 띄우는 `app.py` 대신 `weekly_scan.py`를 사용합니다.

```bash
python weekly_scan.py
```

`weekly_scan.py`는 화면을 띄우지 않고 미국 시가총액 Top 100을 불러온 뒤, 이번 주 진행 중인 주봉 기준 매수후보를 스캔하고 결과 CSV를 `outputs/` 폴더에 저장합니다.

처음 한 번은 필요한 패키지를 설치해야 합니다.

```bash
pip install -r requirements.txt
```

## 사용 방법

1. 상단 검색창에 `GOOG`, `AAPL`, `TSLA` 같은 티커를 입력합니다.
2. 검색 버튼을 누르거나 Enter를 누릅니다.
3. `data/{TICKER}.csv`가 있으면 캐시 파일을 먼저 사용합니다.
4. 캐시 파일이 없거나 읽을 수 없으면 야후파이낸스에서 일봉 데이터를 자동으로 불러온 뒤 월요일 시작 주봉으로 재구성하고 `data/{TICKER}.csv`에 다시 저장합니다.
5. 먼저 야후 차트 API를 직접 호출하고, 실패하면 `yfinance`를 보조로 시도합니다.
6. 화면에는 MMRM 매수후보 표만 표시됩니다.

`BRK.B`처럼 점이 들어간 티커는 화면과 저장 파일에는 그대로 표시하지만, 야후파이낸스 요청에는 `BRK-B`처럼 하이픈 형식으로 자동 변환합니다.

## 화면 구조

Tkinter 기본 스타일의 3단 구조입니다.

* 왼쪽: 미국 시가총액 Top 100 목록
* 가운데 상단: 티커 입력창과 검색 버튼
* 가운데 하단: 개별 종목 MMRM 매수후보 표
* 오른쪽: 이번주 매수후보 스캐너
* Enter 키 검색 지원

현재 UI에서는 전체 계산표 탭을 보여주지 않습니다. 사용자는 매수후보만 보고 싶다고 했기 때문에 화면을 단순하게 유지합니다.

매수후보 표의 첫 번째 컬럼은 `매수포인트날짜`입니다. 예전에는 `Date`와 `ObservationStartDate`를 함께 보여줬지만 헷갈려서 제거했습니다.

이 컬럼명은 기존 구현 이력상 남아 있는 이름입니다. 이 README에서는 해당 컬럼을 MMRM 조건이 포착된 `매수후보 날짜`로 해석합니다.

## 중앙 검색과 오른쪽 스캐너의 차이

중앙 티커 검색은 과거 확정 주봉 확인용입니다.

* 개별 종목 검색은 마감된 과거 주봉 기준으로 MMRM 매수후보를 확인합니다.
* 진행 중인 이번 주 주봉은 제외합니다.
* 기존 캐시 파일을 사용할 수 있습니다.
* 검색 과정에서 야후 데이터를 새로 받아 `data/{TICKER}.csv`를 갱신할 수 있습니다.
* 결과는 `outputs/{TICKER}_buy_points.csv`에 저장합니다.
* 전체 계산표는 디버깅용으로 `outputs/{TICKER}_full_table.csv`에 저장하지만 GUI에는 보여주지 않습니다.

오른쪽 Top100 스캐너는 이번 주 후보 확인용입니다.

* 미국 시가총액 Top 100 종목 전체를 대상으로 이번 주 매수후보를 찾습니다.
* 진행 중인 이번 주 주봉까지 포함해 현재 조건을 확인합니다.
* Top 100 스캐너는 금요일 주봉 마감 직전에 후보를 빠르게 확인하기 위한 기능입니다.
* 각 종목의 데이터를 야후에서 새로 다운로드합니다.
* 기존 CSV를 최신 데이터로 덮어씁니다.
* 이번 주 주봉이 새 MMRM 매수후보로 기록되는 종목만 오른쪽 표에 표시합니다.
* 이미 과거 주봉에서 매수후보가 나온 종목은 이번 주 후보로 표시하지 않습니다.
* 결과는 확정된 과거 매수지점이 아니라 `이번주 매수후보`입니다.
* 금요일 종가 전에 조건이 바뀔 수 있습니다.
* 스캔 결과는 자동 저장하지 않습니다.
* 필요할 때 `스캔 저장하기` 버튼을 눌러 CSV로 저장합니다.

## 미국 시총 Top 100 목록

왼쪽 패널의 `Top 100 불러오기` 버튼을 누르면 StockAnalysis의 미국 상장 종목 시가총액 순위 페이지에서 실시간 목록을 불러옵니다.

출처:

```text
https://stockanalysis.com/list/biggest-companies/
```

StockAnalysis 페이지는 미국에 상장되어 NASDAQ, NYSE, NYSE American에서 거래되는 종목을 시가총액 기준으로 정렬하며, OTC와 비상장사는 포함하지 않는다고 설명합니다.

목록 컬럼:

* 순위
* 티커
* 회사명
* 시가총액

동작:

1. 프로그램 실행 시 Top 100 목록을 자동으로 불러오지 않습니다.
2. 사용자가 `Top 100 불러오기` 버튼을 눌렀을 때만 실시간 조회합니다.
3. 조회 성공 시 왼쪽 표에 상위 100개 종목을 표시합니다.
4. 행을 클릭하면 해당 티커가 검색창에 자동 입력되고 바로 검색됩니다.
5. 조회 실패 시 오류 안내를 띄우고 목록은 비워둡니다.
6. Top 100 목록은 캐시 파일로 저장하지 않습니다.
7. 실시간 조회에 실패하면 과거 목록을 대신 보여주지 않습니다.

## 이번주 매수후보 스캐너

오른쪽 패널의 `Top 100 스캔` 버튼은 미국 시가총액 Top 100 종목 전체를 대상으로 이번 주 MMRM 매수후보를 찾습니다.

이 스캐너는 금요일 주봉 마감 직전에 후보를 빠르게 확인하기 위한 기능입니다. 기존 개별 종목 검색은 마감된 과거 주봉 기준 확인용이고, Top 100 스캐너는 이번 주 진행 중인 주봉까지 포함해 현재 조건을 확인합니다.

동작:

1. `Top 100 스캔` 버튼을 누릅니다.
2. 왼쪽 Top 100 목록이 비어 있으면 인터넷에서 목록을 먼저 불러오고, 왼쪽 표에도 표시합니다.
3. 표시된 Top 100 목록을 기준으로 각 종목의 데이터를 야후에서 새로 다운로드합니다.
4. 이번 주 진행 중인 주봉까지 포함해 Momentum, MACD, RSI, MFI를 계산합니다.
5. 이번 주 주봉이 새 MMRM 매수후보로 기록되는 종목만 오른쪽 표에 표시합니다.
6. 이미 과거 주봉에서 매수후보가 나온 종목은 이번 주 후보로 표시하지 않습니다.
7. 데이터 조회에 실패한 종목은 한 번 더 자동 재시도합니다.
8. 재시도 후에도 실패한 종목은 최종 실패 목록으로 기록합니다.
9. 오른쪽 후보 표에서 행을 클릭하면 가운데 검색창에 티커가 들어가고 바로 개별 검색됩니다.
10. 스캔 결과는 자동 저장하지 않습니다. 필요할 때 `스캔 저장하기` 버튼을 눌러 CSV로 저장합니다.

스캐너 결과 표 컬럼:

* 순위
* 티커
* 회사명
* 시가총액
* 주봉시작일
* 스캔일
* Close
* MACD
* Signal
* Momentum
* RSI
* MFI

`주봉시작일`은 해당 후보가 속한 월요일 시작 주봉의 날짜입니다.

`스캔일`은 사용자가 스캐너를 실행한 날짜입니다.

스캐너는 이번 주 진행 중인 주봉을 사용하므로 결과는 확정된 과거 매수포인트가 아니라 `이번주 매수후보`입니다. 금요일 종가 전에 조건이 바뀔 수 있습니다.

## 출력 파일

검색이 끝나면 아래 파일이 저장됩니다.

```text
outputs/{TICKER}_buy_points.csv
outputs/{TICKER}_full_table.csv
```

Top 100 스캐너 결과는 `스캔 저장하기` 버튼을 눌렀을 때 사용자 다운로드 폴더에 아래 파일로 저장됩니다.

```text
Downloads/top100_scan_candidates_YYYY-MM-DD.csv
Downloads/top100_scan_failures_YYYY-MM-DD.csv
```

`weekly_scan.py`로 화면 없이 자동 스캔을 실행하면 아래 파일이 프로젝트 `outputs/` 폴더에 저장됩니다.

```text
outputs/weekly_scan_candidates_YYYY-MM-DD.csv
outputs/weekly_scan_failures_YYYY-MM-DD.csv
```

CSV는 Excel에서 한글이 잘 보이도록 `utf-8-sig`로 저장합니다.

## 기준 시나리오

MMRM 매수후보는 다음 흐름으로 판정합니다.

1. MACD 하락영역에서 시작된 MACD 상승흐름시작이 발생하면 관찰을 시작합니다.
2. 그 MACD 상승흐름이 유지되는 동안 기다립니다.
3. Momentum > 0, RSI > 50, MFI > 50이 처음 동시에 만족되는 마감 주봉을 매수후보로 기록합니다.
4. 세 조건이 만족되기 전에 MACD 하락흐름시작이 나오면 기존 관찰 상태를 리셋합니다.

### 조건 세부 기준

* MACD 하락영역: MACD선이 0 아래에 있는 영역
* MACD 상승흐름: MACD선 > Signal선
* MACD 하락흐름: Signal선 > MACD선
* MACD 상승흐름시작: 전 주에는 Signal선 > MACD선이었다가 이번 주에 MACD선 > Signal선으로 바뀐 시점
* 하락영역 상승흐름시작: MACD 상승흐름시작이 발생한 주의 MACD가 0 아래이거나, 직전 주 MACD가 0 아래였던 경우. 즉 상승흐름시작 주에 MACD가 0 위로 올라왔더라도 직전 주가 하락영역이면 인정합니다.
* MACD 하락흐름시작: 전 주에는 MACD선 > Signal선이었다가 이번 주에 Signal선 > MACD선으로 바뀐 시점
* Momentum 조건: Momentum > 0
* RSI 조건: RSI > 50
* MFI 조건: MFI > 50

MMRM 매수후보는 `MACD 하락영역에서 시작된 MACD 상승흐름`이 유지되는 동안 세 보조 조건이 처음 동시에 만족되는 주봉입니다.

관찰이 시작된 뒤에는 MACD가 상승영역으로 올라와도 MACD 하락흐름시작이 나오기 전까지 관찰을 유지합니다.

관찰 시작일은 내부 계산에는 사용하지만 화면의 매수후보 표에는 표시하지 않습니다.

## 데이터 기준

프로그램은 야후파이낸스에서 데이터를 자동으로 불러옵니다.

야후의 `1wk` 원본 주봉 날짜가 차트 서비스마다 다르게 보일 수 있어서, 원본 `1wk` 주봉은 사용하지 않습니다.

항상 일봉을 받은 뒤 프로그램 안에서 월요일 시작 주봉으로 재구성합니다.

`data/{TICKER}.csv` 파일은 다음 실행을 빠르게 하기 위한 캐시입니다. 사용자가 직접 CSV를 준비하지 않아도 됩니다.

개별 종목 검색은 기존 캐시를 사용할 수 있습니다.

Top 100 스캐너는 금요일 밤 현재 기준 후보를 찾는 기능이므로 매번 야후에서 새 데이터를 받아 기존 CSV를 최신 데이터로 덮어씁니다.

### 중요한 데이터 결정

처음에는 Yahoo Finance의 `interval=1wk` 데이터를 그대로 사용했습니다.

하지만 MRK 테스트에서 Investing.com 화면과 주봉 캔들 날짜 및 가격이 다르게 나오는 문제가 확인되었습니다.

예시로 Investing.com에서 보이는 MRK 2025-08-18 주봉은 사용자가 본 차트와 맞았지만, Yahoo `1wk` 원본 캐시는 2025-08-21 같은 목요일 라벨과 다른 종가를 만들었습니다.

그래서 현재 방식은 다음과 같습니다.

1. Yahoo chart API에서 일봉 데이터를 받습니다.
2. Yahoo chart API 일봉 다운로드가 실패하면 `yfinance`로 일봉 데이터를 받습니다.
3. 프로그램 안에서 일봉을 월요일 시작 주봉으로 직접 묶습니다.
4. 주봉 OHLCV는 아래 기준으로 만듭니다.

   * Open: 해당 주 첫 거래일 시가
   * High: 해당 주 최고가
   * Low: 해당 주 최저가
   * Close: 해당 주 마지막 거래일 종가
   * Volume: 해당 주 거래량 합계
   * Adj Close: 해당 주 마지막 거래일 수정종가
5. 아직 마감되지 않은 현재 주봉은 개별 검색에서는 제외하고, Top 100 스캐너에서는 포함합니다.
6. 이렇게 만든 주봉 데이터를 `data/{TICKER}.csv`에 캐시합니다.

기존에 저장된 캐시가 월요일 라벨 주봉이 아니면 예전 방식 캐시로 보고 무시한 뒤 새로 다운로드합니다.

이 방식은 Investing.com과 최대한 맞추기 위한 선택입니다.

다만 Yahoo와 Investing.com의 원천 데이터, 배당/분할 보정, 지표 계산식이 다르면 완전히 100% 같지 않을 수 있습니다.

## 지표 계산

* Momentum: `Close - Close.shift(14)`
* MACD: Close 기준 EMA 12, EMA 26, Signal 9
* RSI: 14기간 Wilder 방식
* MFI: 14기간 Typical Price와 거래량 기준

개별 종목 검색의 지표는 마감된 주봉 기준으로 계산합니다.

Top 100 스캐너와 `weekly_scan.py`는 이번 주 진행 중인 주봉까지 포함해 계산합니다.

## 중요한 구현 메모

다음 세션에서 이어 작업할 때 특히 아래를 먼저 확인하세요.

* 이 프로그램은 Investing.com 주봉 화면과 맞는 MMRM 매수후보 검출을 목표로 합니다.
* Yahoo `1wk` 원본 주봉은 그대로 쓰지 않습니다.
* Yahoo 일봉 데이터를 받아 월요일 시작 주봉으로 직접 재구성합니다.
* 화면에는 매수후보 표만 표시합니다.
* 매수후보 표에는 관찰시작일을 표시하지 않습니다.
* `outputs/{TICKER}_full_table.csv`는 디버깅용으로 저장되지만 GUI에는 보여주지 않습니다.
* 미국 시총 Top 100 목록은 StockAnalysis에서 버튼 클릭 시 실시간 조회합니다.
* Top 100 목록은 캐시하지 않습니다. 실패하면 오류를 보여주고 목록은 비웁니다.
* Top 100 목록 행 클릭은 해당 티커 즉시 검색입니다.
* Top 100 스캐너는 이번 주 진행 중인 주봉까지 포함해 새 매수후보만 찾습니다.
* MACD 상승흐름시작 주에 MACD가 0 위로 올라왔더라도 직전 주 MACD가 0 아래였으면 하락영역에서 시작된 상승흐름으로 인정합니다.
* 관찰 시작 후에는 MACD가 상승영역으로 올라가도 하락흐름시작 전까지 계속 관찰합니다.
* Top 100 스캐너는 각 종목 데이터를 매번 새로 다운로드하고 기존 CSV를 덮어씁니다.
* Top 100 스캐너 결과와 실패 종목은 `스캔 저장하기` 버튼을 눌렀을 때 사용자 다운로드 폴더에 날짜별 CSV로 저장합니다.
* `weekly_scan.py`는 GitHub Actions 같은 자동실행 환경에서 쓰는 화면 없는 Top 100 스캐너입니다.
* `weekly_scan.py` 결과와 실패 종목은 프로젝트 `outputs/` 폴더에 날짜별 CSV로 저장합니다.
* 사용자가 “세이브포인트”라고 말하면 Git 커밋을 의미합니다.
* 사용자가 “푸시”라고 말하면 로컬 커밋을 GitHub에 업로드하는 것을 의미합니다.

## GitHub Actions 자동 스캔

이 레포에는 `.github/workflows/weekly-scan.yml` 워크플로가 있습니다.

동작 방식:

* 수동 실행: GitHub 레포의 `Actions` 탭에서 `Weekly Top 100 Buy Candidate Scan` 워크플로를 선택한 뒤 `Run workflow`를 누릅니다.
* 자동 실행: 워크플로 파일에 설정된 cron 시간에 실행됩니다.
* 실행 명령: `python weekly_scan.py`
* 저장 방식: 결과 CSV를 레포에 커밋하지 않고, GitHub Actions artifact로 업로드합니다.
* artifact 이름: `weekly-scan-results`

업로드되는 파일:

```text
outputs/weekly_scan_candidates_YYYY-MM-DD.csv
outputs/weekly_scan_failures_YYYY-MM-DD.csv
```

주의:

* GitHub Actions의 cron은 UTC 기준입니다.
* 스캔 결과 CSV는 GitHub 저장소 파일 목록에 자동 커밋되지 않습니다.
* 결과는 해당 Actions 실행 화면의 `Artifacts` 영역에서 다운로드합니다.
* GitHub 로그인 없이 Actions artifact를 직접 다운로드하기는 어렵습니다.

## 문제 해결

Top 100 목록이 안 불러와질 때:

* 인터넷 연결을 확인합니다.
* `https://stockanalysis.com/list/biggest-companies/` 접속이 가능한지 확인합니다.
* `market_cap_provider.py`의 StockAnalysis 표 파싱 로직을 확인합니다.

티커 검색이 실패할 때:

* Yahoo 계열 데이터 다운로드가 막혔는지 확인합니다.
* 티커가 Yahoo에서 쓰는 형식인지 확인합니다.
* 예: `BRK.B` 같은 티커는 Yahoo에서 `BRK-B` 형태가 필요할 수 있습니다.

pytest가 없을 때:

```bash
pip install -r requirements.txt
python -m pytest
```
